Instructions

          

            Telegram Setup

            1. Create a bot via @BotFather.

            2. Copy the bot token into this panel.

            3. Get your user ID(s) from @userinfobot.

            4. Add allowed IDs, then click Test and Save.

            5. Use Send Test to verify delivery.
          

          

            Discord Setup

            1. Create an app + bot in Discord Developer Portal.

            2. Enable required intents (including Message Content if needed).

            3. Invite bot to your server with message permissions.

            4. Paste bot token; optionally add app/guild/channel IDs.

            5. Click Test, then Save, then Send Test.
          

          

            WhatsApp Cloud API Setup

            1. In Meta Developer dashboard, open WhatsApp Cloud API.

            2. Copy access token and phone number ID into this panel.

            3. Configure webhook verify token/secret (if using inbound webhooks).

            4. Add a test recipient number in E.164 format.

            5. Click Test, Save, then Send Test.

Setup Guide

            

              1. Enable + set a token — any secret string you choose.

              2. Save and restart the gateway to activate.

              3. Access from the internet? Use
              Tailscale (recommended),
              ngrok, or
              Cloudflare Tunnel.
              


              Endpoints:

              POST /hooks/agent — full AI run

              POST /hooks/wake — lightweight nudge

              GET /hooks/status — health check


              Auth: Send token as

              Authorization: Bearer <token>

              or x-prometheus-token: <token>


              Works with: GitHub Actions, Stripe, Vercel, n8n, IFTTT, Home Assistant, or any script.

What is MCP?

              

                Model Context Protocol (by Anthropic) is a standard that lets AI connect to external tools and data sources.


                Each MCP server exposes a set of tools — Prometheus discovers them automatically when you connect.
                The AI can then call them during tasks, just like its built-in tools.


                Servers run locally as child processes (stdio) or connect to remote HTTP endpoints (SSE).
                No data leaves your machine unless the tool itself calls an external API.
