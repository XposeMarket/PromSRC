# Visual Quality Rubric

Score mentally before finalizing. Fix any severe miss.

## Product Truth

- The first viewport shows the real product/app/work surface.
- The UI does not advertise features that are not present.
- The subject is visible when the subject is visual.

## Layout

- Main regions are easy to identify.
- No nested card soup.
- Repeated items use consistent dimensions.
- Important controls do not jump when state changes.
- Mobile layout preserves the primary workflow.

## Typography

- Headings match the scale of their container.
- Body text is readable.
- Labels fit controls.
- No negative letter spacing.
- No viewport-width font scaling.

## Color

- Palette supports hierarchy.
- Accent color is used sparingly.
- Semantic states are consistent.
- The page is not dominated by a generic single hue.
- Contrast is sufficient.

## Controls

- Controls match their job.
- Icon-only actions have labels/tooltips.
- Inputs have labels.
- Focus and disabled states are visible.
- Destructive actions are staged or clearly marked.

## Responsiveness

- No accidental horizontal scroll.
- No overlap or clipping at mobile width.
- Tables/lists degrade intentionally.
- Canvas, board, or media keeps a stable aspect ratio.

## Interaction

- Primary workflow works.
- Empty/loading/error states exist when relevant.
- State changes are visible.
- Console is clean of relevant errors.

## Finish Line

The work is done when a user can try the main workflow without needing an explanation from the assistant.
