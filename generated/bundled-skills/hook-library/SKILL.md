# Hook Library

Use this skill to turn weak openings into **scroll-stopping hooks**.

This is a curated library of psychological hook patterns for social posts, short-form video intros, ads, captions, and opening lines. The goal is **not** to copy templates blindly — it is to choose the right psychological trigger for the user's intent, then adapt the wording so it fits their voice, niche, and platform.

---

## When to Use This

Use this skill when the user asks for:
- a **hook** for a post, video, ad, caption, or thread
- a better **opening line** or **first sentence**
- a more **engaging intro**
- help making content more **attention-grabbing**
- multiple hook options for the same idea
- a hook matched to a platform like **TikTok, Reels, Shorts, Instagram, X, YouTube, ads**

Do **not** use this skill for full long-form article writing, email subject-line optimization, or headline work unless the user's real need is clearly the **opening attention grab**.

---

## Core Operating Rule

Don't just hand over random hooks.

First identify:
1. **The content goal** — clicks, watch time, trust, curiosity, envy, urgency, conversion
2. **The platform** — X, TikTok, Reels, Shorts, Instagram caption, ad creative
3. **The tone** — clean, edgy, founder-style, luxury, funny, direct, educational
4. **The proof level** — can we make a bold claim, or do we need a softer version?

Then choose the category that matches the psychology.

---

## Output Standard

Unless the user asks for only one, default to:
- **8–15 hook options**
- grouped by **2–4 relevant categories**
- each adapted to the user's topic
- mark the **strongest 3**
- if helpful, add a **"safer"** and **"harder-hitting"** version

If the user gives very little context, make a reasonable assumption and still produce options.

## Workflow

### 1. Pick the psychological angle
Choose the best category based on the user's desired reaction.

### 2. Adapt the pattern to the niche
Replace generic references with the user's actual:
- product
- audience
- pain point
- transformation
- platform language

### 3. Tighten the wording
Prefer:
- **specific nouns** over vague claims
- **strong verbs** over passive phrasing
- **short punchy rhythm** over bloated wording
- **spoken-language cadence** for short-form video

### 4. Sanity-check credibility
If a hook makes a claim that sounds fake, either:
- soften it, or
- frame it as a story/opinion/observation instead of a fact

### 5. Match the voice
If the user sounds premium, don't give them sloppy hooks.
If the user sounds edgy, don't give them sanitized corporate copy.

## Response Patterns

### If the user says: "Give me hooks for this"
Return:
1. **Best-fit categories**
2. **8–15 adapted hook options**
3. **Top 3 recommendation**
4. Optional **platform-specific rewrites**

### If the user says: "Make this more engaging"
1. Diagnose why the current opener is weak
2. Rewrite it in **3–5 hook styles**
3. Explain which one is best and why

### If the user says: "Write a hook for my reel / TikTok / short"
Bias toward:
- shorter rhythm
- spoken cadence
- sharper emotional contrast
- cleaner first 3 seconds

### If the user says: "Write hooks for my ad"
Bias toward:
- pain
- interruption
- specificity
- believability
- quick relevance to the target audience

---

## Anti-Patterns

Do **not**:
- dump raw templates with no adaptation when the user gave context
- use fake insider/illegal/NDA framing for brands that need credibility
- make unbelievable income or outcome claims unless the user explicitly wants aggressive stylization
- give 20 versions that all sound the same
- force profanity or hype if the user's voice is cleaner
- confuse **hooks** with full captions, full scripts, or complete posts unless the user asked for those too

---

## Quality Bar

A strong hook should do at least one of these immediately:
- create **curiosity**
- sharpen **stakes**
- imply **status**
- show **contrast**
- promise **transformation**
- trigger **recognition** ("that's me")

If the result sounds generic, vague, or like obvious AI copy, rewrite it shorter and more specific.
