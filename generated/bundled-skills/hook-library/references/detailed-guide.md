# Hook Library

Use this skill to turn weak openings into **scroll-stopping hooks**.

This is a curated library of psychological hook patterns for social posts, short-form video intros, ads, captions, and opening lines. The goal is **not** to copy templates blindly — it is to choose the right psychological trigger for the user's intent, then adapt the wording so it fits their voice, niche, and platform.

The full 30-template list for every category lives in `references/templates.md`. Read that file when you need the full library to pick from; this file holds the decision framework plus a couple of examples per category for quick orientation.

---

## When to Use This

Use this skill when the user asks for:
- a **hook** for a post, video, ad, caption, or thread
- a better **opening line** or **first sentence**
- a more **engaging intro**
- help making content more **attention-grabbing**
- multiple hook options for the same idea
- a hook matched to a platform like **TikTok, Reels, Shorts, Instagram, X, YouTube, ads**

Do **not** use this skill for full long-form article writing, email subject-line optimization, or headline work unless the user's real need is clearly the **opening attention grab**.

---

## Core Operating Rule

Don't just hand over random hooks.

First identify:
1. **The content goal** — clicks, watch time, trust, curiosity, envy, urgency, conversion
2. **The platform** — X, TikTok, Reels, Shorts, Instagram caption, ad creative
3. **The tone** — clean, edgy, founder-style, luxury, funny, direct, educational
4. **The proof level** — can we make a bold claim, or do we need a softer version?

Then choose the category that matches the psychology.

---

## Output Standard

Unless the user asks for only one, default to:
- **8–15 hook options**
- grouped by **2–4 relevant categories**
- each adapted to the user's topic
- mark the **strongest 3**
- if helpful, add a **"safer"** and **"harder-hitting"** version

If the user gives very little context, make a reasonable assumption and still produce options.

## Workflow

### 1. Pick the psychological angle
Choose the best category based on the user's desired reaction. Load `references/templates.md` for the full 30-option list per category.

### 2. Adapt the pattern to the niche
Replace generic references with the user's actual:
- product
- audience
- pain point
- transformation
- platform language

### 3. Tighten the wording
Prefer:
- **specific nouns** over vague claims
- **strong verbs** over passive phrasing
- **short punchy rhythm** over bloated wording
- **spoken-language cadence** for short-form video

### 4. Sanity-check credibility
If a hook makes a claim that sounds fake, either:
- soften it, or
- frame it as a story/opinion/observation instead of a fact

### 5. Match the voice
If the user sounds premium, don't give them sloppy hooks.
If the user sounds edgy, don't give them sanitized corporate copy.

## Hook Categories

### 1. Curious Creator
**Use when:** the goal is curiosity, disbelief, stoppage, or "wait, what?" energy.
**Best for:** demos, tool reveals, surprising facts, weird outcomes, fast scroll-stoppers.
**Examples:** "How is this legal?" / "Wait... how does this even work?" / "This broke my brain..."

### 2. Gatekeeper
**Use when:** the content should feel exclusive, forbidden, insider, or behind-the-scenes.
**Best for:** industry secrets, private tactics, controversial reveals, premium-feeling content.
**Rule:** use carefully. If the user's brand is professional or trust-heavy, soften this category instead of making fake secrecy claims.
**Examples:** "My developer told me never to share this..." / "This got me fired but..."

### 3. Someone Showed Me...
**Use when:** the hook should feel social, trusted, passed-along, or naturally recommended.
**Best for:** tutorials, tools, reviews, recommendations, educational content.
**Example:** "My designer showed me this tool and I genuinely can't go back."

### 4. Transformation Result
**Use when:** the user has a strong before/after, measurable outcome, or identity shift.
**Best for:** productivity wins, client growth, lifestyle changes, business proof.
**Instruction:** expand this category custom for the user's niche — the point is the **dramatic end state**, not the exact template.
**Build formula:** `I went from [before] to [after] because [mechanism]` / `This helped me [specific win] in [specific time]` / `How I [dream outcome] without [common pain]`

### 5. Regret Reveal
**Use when:** the hook should create pain around wasted time, money, effort, or delay.
**Best for:** educational content, lessons learned, tools that replace expensive/slow methods.
**Rule:** keep regret believable. If the proof isn't strong, dial down the numbers.
**Example:** "I wasted $10K before finding this..."

### 6. Comparison
**Use when:** the user wants to position something against a category, old method, or competitor.
**Best for:** product marketing, app/tool content, alternatives, replacement framing.
**Instruction:** if direct competitor naming would be risky or weak, use **old way vs new way** instead.
**Example:** "I tried 15 apps, this is the only one that worked..."

### 7. Unintentional Find
**Use when:** the message should feel low-pressure, accidental, authentic, or serendipitous.
**Best for:** softer discovery content, creator posts, recommendation content, de-salesified positioning.
**Example:** "I wasn't even looking for this but..."

### 8. Ragebait Flex
**Use when:** the user wants aspirational, envy-inducing, status-heavy, lifestyle-forward framing.
**Best for:** creator brands, money content, freedom content, lifestyle proof, automation flexes.
**Rule:** use only when the user's brand can support big energy. Avoid this category for sober B2B trust-building unless they explicitly want aggressive style.
**Example:** "POV: You just made $10K in your sleep..."

### 9. Trend Hijack
**Use when:** the user wants hooks that fit current short-form content formats.
**Best for:** TikTok, Reels, Shorts, trend-based content, creator-style scripting.
**Rule:** don't force stale trend language if it already sounds dated. Use the format, not cringe imitation.
**Example:** "POV trend but it's actually useful this time..."

## Response Patterns

### If the user says: "Give me hooks for this"
Return:
1. **Best-fit categories**
2. **8–15 adapted hook options**
3. **Top 3 recommendation**
4. Optional **platform-specific rewrites**

### If the user says: "Make this more engaging"
1. Diagnose why the current opener is weak
2. Rewrite it in **3–5 hook styles**
3. Explain which one is best and why

### If the user says: "Write a hook for my reel / TikTok / short"
Bias toward:
- shorter rhythm
- spoken cadence
- sharper emotional contrast
- cleaner first 3 seconds

### If the user says: "Write hooks for my ad"
Bias toward:
- pain
- interruption
- specificity
- believability
- quick relevance to the target audience

---

## Anti-Patterns

Do **not**:
- dump raw templates with no adaptation when the user gave context
- use fake insider/illegal/NDA framing for brands that need credibility
- make unbelievable income or outcome claims unless the user explicitly wants aggressive stylization
- give 20 versions that all sound the same
- force profanity or hype if the user's voice is cleaner
- confuse **hooks** with full captions, full scripts, or complete posts unless the user asked for those too

---

## Quality Bar

A strong hook should do at least one of these immediately:
- create **curiosity**
- sharpen **stakes**
- imply **status**
- show **contrast**
- promise **transformation**
- trigger **recognition** ("that's me")

If the result sounds generic, vague, or like obvious AI copy, rewrite it shorter and more specific.
