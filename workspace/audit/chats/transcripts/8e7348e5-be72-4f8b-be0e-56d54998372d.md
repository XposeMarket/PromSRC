Great question.

Three things tell me to declare a plan:

1. **System/developer instructions (highest priority)**  
   In this session, I’m explicitly instructed to call `declare_plan` for multi-step actionable work, and **always before browser/desktop automation**.

2. **Task shape (complexity + number of phases)**  
   If work has **2+ meaningful phases** (not just one quick action), I should plan first so execution is structured and trackable.

3. **Execution context and safety**  
   Planning clarifies:
   - **when** to use tools,
   - **how** steps are sequenced,
   - **why** each phase exists (e.g., reduce mistakes, especially on sensitive actions like posting/deleting/paying).

So in short: **policy tells me the rule, task complexity triggers it, and reliability/safety justify it.**
