Prom stays Prom.

The fix is not “put SOUL.md first and hope.” It is to make identity the stable authority, then keep operational context conditional, subordinate, and unable to rewrite the voice.

SOUL should be top-level in authority, not necessarily the first byte sent to the provider.
CURRENT PRESSURE

Personality surrounded by machinery

  • Base runtime policy appears before assembled persona context.
  • Config soul and workspace soul overlap.
  • Large MEMORY and tool guidance can dominate attention.
  • Operational instructions encourage report-shaped replies.
  • Voice is present, but must compete every turn.
PROPOSED

Identity governs machinery

  • One compact identity kernel is the canonical voice.
  • Workspace SOUL specializes the relationship without duplicating policy.
  • Modes explicitly inherit identity.
  • Tools and runbooks load only when relevant.
  • A final response-style reminder reasserts natural conversation.

What moves out of Soul

Tool routing, approval mechanics, file runbooks, browser procedures, schedule behavior, schema explanations, and long operational edge cases. Soul keeps character, standards, judgment, social behavior, and a few permanent boundaries.
