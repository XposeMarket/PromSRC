90: ## Read so far
91: 
92: - Hermes internal telemetry is working and useful. We can now separate outer wall time from model latency and tool latency.
93: - Hermes is spending a lot of tokens per benchmark because each single run carries a large injected context/config load. `local_web_debug_v1` consumed ~919k input tokens across 25 model calls.
94: - Hermes final-answer compliance is currently a benchmark weakness. It completed much of the actual shell/local-web work but surfaced diffs/review output instead of the required exact final line.
95: - Prometheus remains much faster on the first comparable browser/shell lanes, but Prometheus needs a proper export harness for local-web and desktop if we want the report to be fully normalized.
96: 
97: ## Next steps
98: 
99: 1. Investigate Hermes/Codex CLI diff leakage on `local_web_debug_v1`; prompt-level guard improved shell but did not suppress local-web review diffs.
100: 2. Export Prometheus-native telemetry for `local_web_debug_v1` and `desktop_basic_v1` into the same summary schema.
101: 3. Add derived metrics: orchestration overhead = wall ms - model ms - tool ms, model-call count per pass, tokens per successful benchmark, and final-format compliance.
