9: ## 1) The single assembly funnel
10: 
11: Almost every live agent turn goes through one function:
12: 
13: - **Entry:** `src/gateway/routes/chat.router.ts` — `handleChat()` at line 1756
14: - **Default mode:** `executionMode = 'interactive'` (line 1764)
15: 
16: Each turn's system message is built from **four layers**:
17: 
18: | Layer | Source file | Lines | What it is |
19: |-------|-------------|-------|------------|
20: | 1. Execution-mode block | `src/gateway/routes/chat.router.ts` | 3105–3158 | `EXECUTION MODE: …` per mode |
21: | 2. Core base policy | `src/gateway/routes/chat.router.ts` | 3160–3199 | "You are Prom…", creative/browser/team routing, plan protocol |
22: | 3. Personality / memory / tools | `src/gateway/prompt-context.ts` | 964–1320 | `[USER]`, `[SOUL]`, `[MEMORY]`, `[TOOLS]`, skills, CIS, retrieved memory |
23: | 4. Per-run caller overlay | various (see §4) | — | task context, team dispatch, schedule owner, boot, etc. |
24: 
25: Final system-prompt assembly (`buildSystemPrompt`):
26: 
27: - **File:** `src/gateway/routes/chat.router.ts`
28: - **Lines:** 3201–3212
29: 
30: **Ordering difference (important):**
31: 
32: - **`team_subagent`:** `baseSystemPrompt` → model block → recent tool log → **personalityCtx** → **callerContext** → browser state
33: - **Everything else:** `baseSystemPrompt` → model block → recent tool log → **callerContext** → browser state → **personalityCtx** → onboarding block (if applicable)
34: 
35: `buildPersonalityContext()` is called at `chat.router.ts:3070` before `buildSystemPrompt()`.
