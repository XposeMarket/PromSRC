49: ## 21) Model Configuration and Presets
50: 
51: Current default LLM config in source:
52: 
53: - active provider default: `ollama`
54: - default Ollama model: `qwen3:4b`
55: - default OpenAI Codex model: `gpt-5.5`
56: - default OpenAI API model: `gpt-5.5`
57: - OpenAI Codex and OpenAI API static model lists expose the official GPT-5.6 preview model IDs published by OpenAI: `gpt-5.6-sol` (flagship), `gpt-5.6-terra` (balanced), and `gpt-5.6-luna` (fast/low-cost). Do not expose invented aliases such as `gpt-5.6`, `gpt-5.6-codex`, `gpt-5.6-codex-mini`, or plain `sol`; OpenAI's preview help article lists only the three fully-qualified IDs for API and Codex. GPT-5.6 preview access is separately provisioned for approved API organizations and Codex workspaces, so `gpt-5.5` remains the safe default/fallback until the active account is enabled.
58: - Official GPT-5.6 preview pricing per 1M tokens: Sol $5 input / $30 output, Terra $2.50 input / $15 output, Luna $1 input / $6 output. GPT-5.6 cache writes are billed at 1.25x uncached input and cache reads keep the 90% cached-input discount.

93: Other current model facts:
94: 
95: - `switch_model` is turn-scoped, not a global config save
96: - its exposed tiers are `low` and `medium`
97: - OpenAI, OpenAI Codex, and Perplexity provider config support `reasoning_effort`
98: - validated reasoning efforts include `none`, `minimal`, `low`, `medium`, `high`, `xhigh`, and `max`; GPT-5.6 introduces `max` reasoning effort, while older providers/models may normalize or ignore unsupported values.
99: - Anthropic provider config supports `extended_thinking` and `thinking_budget`
100: 
101: Provider-aware context budgeting lives in `src/gateway/context/model-context.ts`.
102: 
103: Current context-profile behavior:
104: 
105: - `resolveActiveModelContextProfile(...)` reads the active provider/model from config and returns `providerId`, `model`, `contextWindowTokens`, `maxOutputTokens`, tokenizer family, reasoning-token support, reasoning budget, and source
106: - profile sources are `config_override`, `provider_metadata`, `known_table`, `ollama_num_ctx`, or `fallback`
107: - provider config overrides can set `context_window` / `contextWindowTokens`, `max_output_tokens` / `maxOutputTokens`, `reasoning_budget_tokens` / `thinking_budget`, and `tokenizer`
108: - provider metadata can supply `contextWindowTokens`, `maxOutputTokens`, and tokenizer when available
109: - Ollama context can come from configured `num_ctx`, `LOCALCLAW_SESSION_NUM_CTX`, or `LOCALCLAW_CHAT_NUM_CTX`
110: - tokenizer families are `openai`, `anthropic`, `gemini`, `llama`, `qwen`, and `heuristic`
111: 
112: Known context table facts:
113: 
114: - OpenAI/OpenAI Codex `gpt-5*` and Codex-named models are currently treated as 400k context with 128k max output
115: - OpenAI/OpenAI Codex `gpt-4.1*` is treated as about 1,047,576 context with 32,768 max output
116: - OpenAI/OpenAI Codex `gpt-4o` and `o*` models are treated as 128k context
117: - Anthropic `claude-opus-4-8` is treated as 1m context with 128k max output
118: - Anthropic `claude-*` models are treated as 200k context
119: - Gemini `gemini-*` models are treated as 1m context
120: - xAI static models include `grok-build-0.1`, the OAuth/subscription-gated `grok-composer-2.5-fast` Composer 2.5 id, and current Grok 4.x ids. Composer 2.5 is available in Grok Build to SuperGrok/X Premium+ and may be absent from public `/v1/models`, so keep it as a curated static model while live model discovery remains enabled.
121: - Perplexity sonar models and xAI Grok/Composer models currently fall back to 128k known-table entries unless overridden or provider metadata says otherwise
122: - unknown local/llama providers fall back lower, currently 8192; unknown cloud-ish providers fall back to 32768
123: 
124: Budget math:
125: 
126: - `buildContextBudget(...)` reserves output tokens, reasoning tokens when supported, and 10% safety headroom
127: - usable input budget is context minus reserved output, reasoning, and headroom
128: - compaction triggers at 75% of the usable input budget
129: - recent tool context budget is about 16% of usable input budget, with a 600-token floor
130: - summary budget is about 8% of usable input budget, with a 700-token floor
131: 
132: Token counts are estimates, not provider-billed truth. `estimateTextTokensForModel(...)` uses character-density heuristics by tokenizer family and gives denser code/log text a smaller divisor. Provider usage metadata should be used for calibration where available, but UI/context decisions should still tolerate approximation.
133: 
134: Anthropic extended-thinking request shape:
135: 
136: - Opus 4.7 and newer Opus aliases use adaptive thinking (`thinking: { type: "adaptive" }`) with high effort output config instead of legacy `budget_tokens`.
137: - Older supported Claude thinking models continue to use the configured `thinking_budget`.
138: - The native Anthropic extension does not carry a descriptor-level static model list; `AnthropicAdapter.listModels()` returns the known native model set. Aggregator descriptors keep static fallback lists in their provider-specific ID formats.
