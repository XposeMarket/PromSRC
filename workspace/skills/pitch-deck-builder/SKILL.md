# Pitch decks

Build one coherent argument for one audience and decision.

1. Confirm audience, decision, stage, time limit, ask, evidence, and brand constraints.
2. Write the narrative spine before individual slides.
3. Give each slide one job, one takeaway title, and only the evidence needed for that takeaway.
4. Sequence context/problem, insight, solution, proof, model/plan, differentiation, risks, and ask according to deck type.
5. Replace unsupported claims with evidence, assumptions, or explicit validation gaps.
6. Review pacing, visual hierarchy, consistency, source notes, and whether the ask follows from the argument.

Do not overload slides, invent traction, or treat a standard startup template as mandatory.

Read [detailed-guide.md](references/detailed-guide.md) for deck-specific structures, slide prompts, metrics, critique rubrics, and handoff formats.
