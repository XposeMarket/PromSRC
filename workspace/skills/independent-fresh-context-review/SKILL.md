# Independent Fresh-Context Review

Review consequential work without inheriting implementation reasoning.

The producer creates `review-brief.md` with objective, criteria, artifacts, decisions, doubts, tests, and evidence paths, but not persuasive hidden reasoning. Start a fresh reviewer context. Restate the contract, attack likely-wrong assumptions, and inspect real files, UI, logs, tests, and outputs. Label every finding confirmed, disproven, or unresolved with evidence. Rank actionable findings and return exactly one verdict: verified, needs remediation, or blocked. A reviewer who implemented the artifact is not independent. After remediation, re-check affected criteria. Never return vague 'looks good'.
