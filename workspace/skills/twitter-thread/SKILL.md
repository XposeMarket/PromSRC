# X/Twitter Thread

Turn the supplied idea, article, research, story, or draft into a coherent sequence. Do not invent facts or engagement claims.

## Structure

1. Write a specific first-post hook that earns the continuation without saying “a thread.”
2. Establish context and why it matters in one or two posts.
3. Give each body post one useful idea that can stand alone; use concrete examples or sourced data when available.
4. Add a recap only if it helps retention rather than repeating every line.
5. End with one appropriate call to action, or none when the user did not request promotion.

Keep each standard post within the platform limit and report any exceptions. Use readable line breaks, restrained emoji, accurate numbering, and consistent voice. Do not manufacture polarizing claims, false scarcity, personal experience, or research counts.

Deliver as `[1/N]`, `[2/N]`, and so on, followed by a brief character-limit check. For detailed hook patterns, body templates, topic structures, recap/CTA patterns, and optional browser posting order, read [the detailed guide](references/detailed-guide.md).

Posting is a separate external action. Use the X browser workflow only when explicitly requested, confirm each post before chaining replies, and never treat a draft request as authorization to publish.
