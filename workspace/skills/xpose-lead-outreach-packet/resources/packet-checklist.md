# Xpose Manual Outreach Packet Checklist

Use this checklist when packaging an already-qualified Xpose Market lead into a manual review/outreach packet.

## 1. Preconditions

Proceed only if the lead has:

- [ ] Qualification scorecard or master scorecard entry.
- [ ] At least one support artifact: audit, intel, pitch, demo, brand kit, or verified public source.
- [ ] Clear Xpose value angle: conversion, trust packaging, mobile CTA, local SEO, visual credibility, service-page clarity, reputation leverage, or funnel simplification.

Stop and write a gap list if the lead does not have enough evidence.

## 2. Input discovery

Read or locate:

- [ ] Master scorecard: `teams/team_moto00fr_2c910f/workspace/xpose-market/qualified/master-scorecard.md`
- [ ] Lead qualification file: `teams/team_moto00fr_2c910f/workspace/xpose-market/qualified/<lead-slug>-qualification.md`
- [ ] Website audit: `teams/team_moto00fr_2c910f/workspace/xpose-market/audits/...`
- [ ] Business intel: `teams/team_moto00fr_2c910f/workspace/xpose-market/intel/<lead-slug>-intel.md`
- [ ] Pitch package: `teams/team_moto00fr_2c910f/workspace/xpose-market/pitches/<lead-slug>-pitch.md`
- [ ] Demo folder: `teams/team_moto00fr_2c910f/workspace/xpose-market/demos/<lead-slug>/`
- [ ] Brand assets: `teams/team_moto00fr_2c910f/workspace/xpose-market/brand-assets/<lead-slug>/` and/or `generated/images/brand-kits/...`
- [ ] Existing packet examples, especially `xpose-market/outreach-packets/frederick-roof-repair/`

If public evidence is stale and the approved task allows it, refresh with web search/fetch. Otherwise use saved artifacts and label caveats.

## 3. Evidence rules

For every business claim, attach or imply a supporting artifact/source:

- [ ] Rating/review count comes from scorecard, intel, audit, or refreshed source.
- [ ] Website problem claims come from audit/demo notes or direct inspected evidence.
- [ ] Business age, owner name, services, service area, and social proof are supported.
- [ ] Offer angle follows the evidence, not generic sales language.
- [ ] Missing facts are marked unknown or omitted.

Never fabricate:

- [ ] Screenshots/mockups that do not exist.
- [ ] Review counts or ratings.
- [ ] Owner/contact names.
- [ ] Revenue impact metrics.
- [ ] Testimonials or customer quotes.
- [ ] Competitor comparisons not in evidence.

## 4. Output folder convention

Create:

```text
xpose-market/outreach-packets/<lead-slug>/
├── manual-outreach-packet.md
├── one-page-demo.md
└── README.md
```

Use lowercase kebab-case for `<lead-slug>`.

## 5. `manual-outreach-packet.md` checklist

Include:

- [ ] Title with business name and “Manual Outreach Packet”.
- [ ] Prepared date.
- [ ] Lead grade/status if known.
- [ ] Clear warning: **DO NOT SEND AUTOMATICALLY**.
- [ ] Business Snapshot table.
- [ ] Why this is an Xpose-fit lead.
- [ ] Evidence-backed problem teardown.
- [ ] Before/after narrative.
- [ ] Demo reference and brand/visual asset references if available.
- [ ] Value proposition / what Raul should say.
- [ ] Email draft for manual use.
- [ ] SMS/DM-style short draft for manual use.
- [ ] Call opener or voicemail draft for manual use.
- [ ] Follow-up draft for manual use.
- [ ] Objections and answers.
- [ ] Supporting artifact paths.
- [ ] Critical status section: no outreach/contact/CRM automation.
- [ ] Raul manual decision checklist.
- [ ] Summary and next manual step.

## 6. `one-page-demo.md` checklist

Include:

- [ ] The Lead.
- [ ] The Problem.
- [ ] The Opportunity.
- [ ] The Pitch in one short paragraph.
- [ ] Expected impact, framed directionally and without guarantees.
- [ ] Xpose Market service scope.
- [ ] Quick facts table.
- [ ] Supporting assets.
- [ ] Manual-review/no-contact status.

## 7. `README.md` checklist

Include:

- [ ] What is in the packet.
- [ ] Quick start for Raul.
- [ ] Supporting artifacts and analysis index.
- [ ] Demo/visual asset index.
- [ ] Live business links only if already in artifacts or freshly verified.
- [ ] Key context and why this lead is qualified.
- [ ] How to use for email/call/screen share/manual LinkedIn or DM.
- [ ] Status and ground rules.
- [ ] File structure.
- [ ] Methodology and confidence.
- [ ] Caveats.
- [ ] Next steps.

## 8. No-contact QA gate

Before completion, confirm:

- [ ] No email was sent.
- [ ] No contact form was submitted.
- [ ] No phone call was placed.
- [ ] No SMS was sent.
- [ ] No DM or social post was sent.
- [ ] No CRM record was created.
- [ ] No follow-up automation was scheduled.
- [ ] All outreach copy is clearly draft/manual-use only.
- [ ] Raul remains the human decision-maker for if/when/how to contact the lead.

## 9. Final quality gate

Before reporting done:

- [ ] File paths are correct.
- [ ] Missing assets are listed as gaps.
- [ ] Claims are sourced or caveated.
- [ ] Tone is useful, direct, and sales-ready without hype.
- [ ] Packet is reviewable without needing to inspect every upstream file.
- [ ] No unsupported guarantees or pressure tactics.
- [ ] No external outreach actions occurred.

## 10. Frederick reference pattern

Use as structure reference only:

- `xpose-market/outreach-packets/frederick-roof-repair/manual-outreach-packet.md`
- `xpose-market/outreach-packets/frederick-roof-repair/one-page-demo.md`
- `xpose-market/outreach-packets/frederick-roof-repair/README.md`

Frederick’s successful shape: business snapshot → problem teardown → before/after demo narrative → value proposition → manual outreach variants → objections → supporting artifact paths → no-contact status → next manual step.
